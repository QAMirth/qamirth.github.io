What is TestNG?

TestNG is a testing framework created by Cédric Beust that helps meet a significant number of AQA & Dev testing needs. Namely, TestNG gives the developer the ability to create more flexible and powerful tests. TestNG is widely used together with Selenium.


What does NG mean?

It means "Next Generation". TestNG is similar to JUnit, but it is more efficient when it comes to managing the execution flow of your application. The architecture of the framework helps us make our tests more structured and provide better validation points.


Some notable features of TestNG are:

Powerful and diverse annotations to support test cases. TestNG uses more features of Java and OOP Separates compile-time test code and run-time data configuration information. Parallel execution of tests, use of dependencies between tests. Let's say load and partial failure testing. Flexibility to perform tests with different data sets, through the TestNG.xml file or through the concept of data providers (data-provider). Grouping and prioritization of test cases. Generation of HTML reports, installation of additional software from various plugins. A flexible API plugin is especially important. Generation of test execution logs. Easy integration with Eclipse, Maven, Jenkins, etc. If you are planning to dive into the field of automation testing, TestNG might be your best choice.


  • Annotations.
  • Run your tests in arbitrarily big thread pools with various policies available (all methods in their own thread, one thread per test class, etc…​).
  • Test that your code is multithread safe.
  • Flexible test configuration.
  • Support for data-driven testing (with @DataProvider).
  • Support for parameters.
  • Powerful execution model (no more TestSuite).
  • Supported by a variety of tools and plug-ins (Eclipse, IDEA, Maven, etc…​).
  • Embeds BeanShell for further flexibility.
  • Default JDK functions for runtime and logging (no dependencies).
  • Dependent methods for application server testing.

TestNG is designed to cover all categories of tests: unit, functional, end-to-end, integration, etc…​
