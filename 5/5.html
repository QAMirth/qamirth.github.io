Google Lighthouse

Lighthouse is an open-source, automated tool for improving the performance, quality, and correctness of your web apps. When auditing a page, Lighthouse runs a barrage of tests against the page, and then generates a report on how well the page did.


From here you can use the failing tests as indicators on what you can do to improve your app.

* Quick-start guide on using Lighthouse: https://developers.google.com/web/tools/lighthouse/

* View and share reports online: https://googlechrome.github.io/lighthouse/viewer/

* Github source and details: https://github.com/GoogleChrome/lighthouse


What parameters does Lighthouse evaluate?

Lighthouse analyzes four metrics: performance, accessibility, SEO, and best practices. For progressive web applications, a fifth parameter is added — PWA.


Performance — performance. Analyzes the site loading speed. This assessment is affected by the time it takes to block, render styles, load interactive elements, fonts, and content.


Progressive Web App — Progressive web applications. Checks whether the site registers Service Workers, works offline, and returns a 200 error.


Best Practices — best practices. Checks the site's security and the use of modern web development standards. The assessment depends on whether the site uses HTTPS, outdated APIs, correct encoding, and other parameters.


Accessibility — availability. Checks whether all users can access the content and effectively navigate the site. This assessment depends on the clarity and perceptibility of the content, the ability to control the interface, and navigate the content without using the mouse.


SEO — evaluates the page's compliance with Google's search engine optimization recommendations. Here, the use of meta tags, access to indexing and crawling by robots, the presence of alt attributes for images, adaptation to mobile screens and other characteristics are checked. Each parameter is assessed on a 100-point scale: the higher, the better.


Each group of assessments also has its own color. Green is set at 90-100 points, it shows that everything is fine with the site. Orange can be obtained at 50-89 points. That is, everything is fine with the site, but it can be done even better. If the score is below 49 points, it turns red. This means that the performance needs to be worked on.


A big advantage of Lighthouse is that you can check the quality of the site on both the desktop and mobile versions.

JMeter

Apache JMeter is a load testing tool that is an open source desktop application based on Java. JMeter helps you determine whether the web application under test can handle high load requirements or not. It also helps you analyze the performance of the entire server under high load.


JMeter helps test both static and dynamic resources, helps detect users who are simultaneously present on the site, and includes visual tools to provide analytics. Web application performance testing with JMeter includes load tests and stress tests.


This tool offers the following advantages in performance testing:

  • JMeter can be used to test the performance of both static resources such as JavaScript and HTML, and dynamic resources such as JSP, servlets, and AJAX.
  • JMeter can be used to determine the maximum number of users per unit of time that the site under test can handle.
  • JMeter provides a variety of visual reports on the performance analysis being performed.


Performance testing with JMeter includes:

  • Load testing: Modeling the expected usage of a web application by simulating multiple users accessing it simultaneously.
  • Stress testing: Each web server has a maximum load limit. When the load exceeds the limit, the web server starts responding slowly and throwing errors. The purpose of stress testing is to determine the maximum load that a web server can handle.

Tsung

Tsung is a distributed load testing system written in Erlang. It claims to support HTTP, WebDAV, SOAP, PostgreSQL, MySQL, LDAP and XMPP/Jabber. Tsung is a completely free and open-source tool, licensed under the GPL (General Public License). There are no paid or trial versions, making it accessible to anyone looking to perform distributed load testing without incurring costs.


Overview and Features: Tsung is a powerful and flexible load testing tool, designed to simulate large numbers of users to test the scalability and performance of web applications, databases, and other services. Written in Erlang, Tsung is capable of generating a very high load due to Erlang's ability to handle thousands of concurrent processes.


Supported Protocols: Tsung supports multiple protocols, including:

  • HTTP/HTTPS: For web applications.
  • WebDAV: An extension of HTTP for managing documents on a remote server.
  • SOAP: For testing web services.
  • PostgreSQL and MySQL: For database load testing.
  • LDAP: For directory services.
  • XMPP/Jabber: For instant messaging protocols.


Advantages:

  • Scalability: Tsung can simulate thousands of simultaneous users, making it suitable for testing large-scale systems.
  • Distributed Load Testing: It allows the distribution of the load across multiple machines, enabling the simulation of even higher traffic.
  • Open-source and Free: As a free tool, it is accessible to developers, testers, and companies of all sizes.
  • Customizable: With its XML configuration files, Tsung offers extensive customization, allowing users to define complex user scenarios and performance metrics.
  • Real-time Monitoring: Tsung provides real-time statistics and graphs, allowing users to monitor the test progress and analyze the results immediately.


Disadvantages:

  • Complexity: Due to its reliance on Erlang and the command line, Tsung has a steep learning curve, particularly for users unfamiliar with these technologies.
  • Limited GUI Support: Tsung primarily relies on CLI and XML configuration, with minimal graphical user interface (GUI) support, which may be less intuitive for users accustomed to GUI-based tools.
  • Less Popularity: Compared to other tools like Apache JMeter or LoadRunner, Tsung has a smaller user community and fewer resources available for troubleshooting and learning.


Principle of Operation: Tsung operates by defining test scenarios in XML files, where users specify the types of requests to send, the number of virtual users to simulate, and the timing of these requests. The tool then executes these scenarios, either on a single machine or distributed across multiple machines. Tsung gathers data on response times, error rates, and throughput, which can be visualized in real-time through its monitoring interface or analyzed post-test.


What Tsung Can and Cannot Do:

  • Can Do:
  • Simulate high concurrency with thousands of users.
  • Perform distributed load testing across multiple machines.
  • Support a wide range of protocols beyond HTTP, including database and messaging protocols.
  • Provide real-time monitoring and detailed logs for in-depth analysis.
  • Cannot Do:
  • Tsung is not as user-friendly as GUI-based tools like JMeter, making it less accessible to beginners.
  • It lacks built-in support for modern web protocols like WebSocket (although it can be extended for this).
  • It does not offer cloud-based load testing out of the box, unlike some commercial tools.

Comparison with Other Tools:

  • Vs. Apache JMeter: While JMeter has a user-friendly GUI and is more popular, Tsung can handle larger numbers of concurrent users with lower resource consumption, thanks to Erlang's efficiency.
  • Vs. LoadRunner: LoadRunner is a paid tool with comprehensive features and support, but Tsung provides a cost-effective alternative for similar load testing scenarios.
  • Vs. Gatling: Gatling, like Tsung, is open-source and focuses on high performance but is written in Scala, offering different scripting capabilities and a more modern approach to load testing.

LoadRunner

LoadRunner, developed by Micro Focus, is one of the most established and widely used performance testing tools on the market. It allows testers to simulate user activity across various applications and protocols, helping to identify and diagnose performance issues before they impact end users.


Supported Protocols: LoadRunner supports a wide array of protocols, making it suitable for testing various types of applications:

  • HTTP/HTTPS: For web applications.
  • Web Services (SOAP, REST): For service-oriented architectures.
  • Database Protocols: Includes Oracle, MS SQL, and others.
  • ERP/CRM Protocols: SAP, Siebel, Oracle E-Business.
  • Other Protocols: FTP, Citrix, LDAP, and more.


Advantages:

  • Comprehensive Protocol Support: LoadRunner supports over 50 protocols, making it versatile for testing a wide range of applications.
  • Realistic User Simulation: It provides sophisticated options for simulating user behavior, including complex scenarios and load patterns.
  • Integration with CI/CD: LoadRunner integrates well with continuous integration and continuous deployment pipelines, making it a good fit for DevOps environments.
  • Robust Analysis Tools: The tool offers detailed performance metrics and analysis features, including bottleneck identification and root cause analysis.
  • Scalability: LoadRunner can simulate thousands of users, either from on-premises machines or from the cloud, to test how applications perform under extreme load conditions.


Disadvantages:

  • Cost: LoadRunner is relatively expensive, which can be a barrier for small businesses or individual developers.
  • Complexity: Due to its wide range of features, LoadRunner has a steep learning curve, and it may require significant time to master.
  • Resource Intensive: Running extensive load tests can require substantial system resources, particularly when simulating large numbers of users.


Principle of Operation: LoadRunner operates by recording user interactions with an application and then replaying these interactions to simulate the behavior of multiple users. The tool includes several components:

  • VuGen (Virtual User Generator): Used to create test scripts by recording actions.
  • Controller: Manages and executes load tests, specifying how virtual users (Vusers) should behave.
  • Analysis: Provides detailed reports and visualizations of test results, helping to identify performance issues.


What LoadRunner Can and Cannot Do:

  • Can Do:
  • Simulate complex user scenarios across multiple protocols.
  • Provide detailed analysis of performance metrics, including response times, throughput, and resource utilization.
  • Integrate with CI/CD pipelines for automated performance testing.
  • Offer cloud-based testing options to simulate large-scale load from geographically distributed locations.
  • Cannot Do:
  • It may not be the best choice for small-scale projects due to its cost and complexity.
  • Does not support all the latest web technologies out of the box; some customization or additional tools may be required.

Comparison with Other Tools:

  • Vs. Apache JMeter: While JMeter is free and open-source, LoadRunner offers more comprehensive protocol support and enterprise-level features, but at a higher cost.
  • Vs. Tsung: Tsung is free and scalable but is less user-friendly and lacks the extensive protocol support and advanced analysis features of LoadRunner.
  • Vs. Gatling: Gatling is also open-source and focused on HTTP protocol testing with a modern approach, whereas LoadRunner offers a broader range of testing capabilities across multiple protocols and more advanced analysis tools.
