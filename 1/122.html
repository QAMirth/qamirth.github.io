The first stage of the software testing life cycle is requirements analysis. At this stage, static testing of the hypothesis is performed by validating the technical documentation of the new release. In our team, such validation is performed in several iterations: the QA team is involved in the first meeting to discuss a certain hypothesis, where the preliminary validation of the idea for implementation in the next release of the application takes place. A QA Engineer points out a corner case or an overlooked feature when designing a hypothesis, and thus helps the Product Manager learn more context about the technical side of creating a feature. At this stage, the QA Engineer is already starting to analyze the technical task, form a test plan and determine the user cases that will need to be tested in the future release. Another advantage of including a testing team is that at such an early stage, the QA Engineer already pre-estimates the labor costs for the current release and can clearly form the risks related to the execution of the technical task.
